What is Monolithic Architecture?

A monolithic architecture refers to a single indivisible unit usually consists of client-side UI, server-side application and database.

  • Web Server
  • Application Server
  • SMTP Server
  • Database Server

Pros

  • Simplify the development
  • Simplify the deployment
  • Better performance
  • Less cross-cutting concerns

Cons

  • Limited Agility
  • Difficult to add new technology
  • Code bloated, IDE gets overloaded
  • Scaling is expensive
  • Simplify development and deployment - developers donâ€™t need to deploy changes or updates separately
  • Better performance - if built properly, monolithic apps are usually more performant than microservice-based apps.
  • Less cross-cutting concerns - cross-cutting concerns, such as audit trails, logging, rate limiting, etc.

  • Limited Agility - every small update requires a full redeployment

Monolithic is perfect for startups that need to get a product up and running ASAP.

What is SOA?

Service-oriented architecture is an application composed of discrete and loosely coupled service that perform a required function.

Pros

  • Service reusability
  • High Reliability
  • Multiple development
  • Better maintainability

Cons

  • Investment cost is higher
  • More overload
  • Complexity increase
  • Service reusability - loosely coupled, reuse in multiple application without influencing other services.
  • High Reliability - easier to debug and test than a huge chunks of code like in the monolithic
  • Multiple development - since consists of layers, independent services can be developed in parallel
  • Better maintainability - since it is independent unit, it is easy to update and maintain without affecting other services.

  • Investment cost is higher - upfront investment of human resources, technology and development
  • More overload - when using multiple services, this increases response time and decreases overall performance
  • Complexity increase - challenging to manage all services

SOA approach is best suited for complex enterprise systems such as banks. Banking system is extremely hard to break into microservices. Monolithc approach also isn't good for it since whole app could be affected.

What is Microservice Architecture?

Microservice is a type of service-oriented software architecture that composes of series of autonomous and independent components that are glued together with APIs.

Pros

  • Easy to develop and deploy
  • Easy test
  • Increase Agility
  • Horizontally Scalable

Cons

  • Complex
  • Concerns in security
  • Programming language switch

Microservices are good, but not for all types of apps. This pattern works great for evolving applications and complex systems.

What is Serverless Architecture?

Serverless architecture is a cloud computing approach in building application without managing infrastructures.

Pros

  • Scalability
  • Reduce costs
  • Easy deployment

Cons

  • Not good for long term task
  • Vendor lock-in

Serverless architecture is beneficial for accomplishing one-time tasks and auxiliary processes.
